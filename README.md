# 企业级私有化 RAG 知识库问答系统

基于本地大模型的完全离线 RAG 系统，支持混合检索、问答溯源、Docker 一键部署。

---

## 📌 项目定位

本项目是一个**企业级离线私有化知识库问答系统**，专为解决数据安全和内部文档智能问答场景设计。
核心特点是：

- 🔒 **完全离线运行**：不依赖任何云端 API，数据不出内网
- 🧠 **本地大模型**：基于 Ollama + DeepSeek 8B，无需 GPU 也可运行
- 🔍 **混合检索**：向量检索 + BM25 关键词检索，兼顾语义和精确匹配
- 📎 **问答溯源**：每个回答都附带原文出处，可追溯可验证
- 🐳 **Docker 部署**：一键启动，开箱即用

## 🚀 快速开始

### 方式一：本地运行

**1. 安装依赖**

pip install -r requirements.txt

2. 启动 Ollama 服务

ollama serve

3. 下载模型（首次运行）

ollama pull deepseek-r1:8b

4. 启动 API 服务

uvicorn rag_api:app --reload

5. 启动前端界面（新开终端）

streamlit run app.py

6. 访问

· API 文档：http://localhost:8000/docs
· 前端界面：http://localhost:8501

---
### 方式二：Docker 部署

 构建镜像
docker build -t rag-qa-system .

 运行容器
docker run -p 8000:8000 rag-qa-system

---
## ⚠️ 注意事项

- **首次运行**：Ollama 会自动下载 deepseek-r1:8b 模型（约 4.7GB），请确保网络稳定，下载时间取决于网速。
- **Embedding 模型**：bge-small-zh-v1.5 模型文件需放在项目根目录的 `bge-small-zh-v1.5/` 文件夹下，否则向量检索会报错。
- **端口占用**：如果 8000 或 8501 端口被占用，启动时会报错，请先释放端口或修改启动命令中的端口号。

🔄 RAG 完整流程

```
用户上传文档
    ↓
文档解析（TXT / PDF）
    ↓
文本分块（chunk_size=500, overlap=50）
    ↓
向量检索（语义匹配） + BM25 检索（关键词匹配）
    ↓
混合检索结果合并去重
    ↓
本地大模型生成回答（DeepSeek 8B）
    ↓
返回答案 + 引用原文出处
```

## 📝 开发笔记

### 遇到的主要问题

1. **Ollama 模型下载失败**
   - 问题：首次拉取 deepseek-r1:8b 时连接超时
   - 解决：配置镜像源 `OLLAMA_HOST=https://ollama.modelscope.cn` 后重新拉取

2. **HuggingFace Embedding 模型无法联网加载**
   - 问题：网络限制导致无法从 HuggingFace 下载 bge-small-zh
   - 解决：从 HuggingFace 官网手动下载模型文件，放入本地目录，代码中改用本地路径加载

3. **混合检索合并去重逻辑**
   - 问题：向量检索和 BM25 返回的结果有重复，直接拼接会导致重复内容进入上下文
   - 解决：使用 `set()` 去重后再拼接，避免重复片段影响回答质量

### 待优化方向

- 替换 Chroma 为 Milvus，支持更大规模的知识库
- 接入 Rerank 模型，对检索结果做二次排序
- 增加用户权限管理，支持多用户独立使用

## 💬 面试常见问题

**Q：为什么用 Chroma 而不是 Milvus？**

Chroma 轻量级，适合快速验证和中小规模知识库。如果数据量超过百万级，我会迁移到 Milvus，它支持分布式索引和更复杂的检索策略。

**Q：混合检索是怎么实现的？**

向量检索负责语义匹配，BM25 负责关键词匹配。两者各自返回 Top 3 结果，合并去重后送入大模型，兼顾语义理解和精确匹配。

**Q：如何降低大模型幻觉？**

- 检索阶段：通过混合检索提高召回质量
- 生成阶段：强制要求模型基于提供的文档内容回答，禁止编造
- 溯源阶段：返回引用原文，用户可以自行核对

**Q：Docker 部署有什么好处？**

将环境和代码打包成镜像，任何机器上只需执行 `docker run` 即可运行，无需手动安装 Python、依赖和模型，适合企业私有化交付。
